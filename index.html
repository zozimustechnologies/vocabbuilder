🚀 Project Overview

VocabBuilder is a Manifest V3 browser extension for Microsoft Edge and Google Chrome that helps users build vocabulary through spaced repetition and daily habit formation. It's designed for competitive exam aspirants (GMAT, GRE, LSAT, CAT), ESL learners, professionals, and lifelong language enthusiasts.

Key Characteristics

  • Platform: Microsoft Edge & Google Chrome (Manifest V3)
  • UI Location: Browser side panel (not a popup)
  • Data Storage: 100% local via Chrome Storage API
  • No Build Step: Pure HTML, CSS, and vanilla JavaScript
  • Version: 0.2.0
  • License: MIT

Tech Stack

Component Technology
Frontend UI HTML5, CSS3 (vanilla, no frameworks)
JavaScript ES6+ vanilla (no dependencies)
Background Service Service Worker (Manifest V3)
Data Persistence Chrome Storage API (chrome.storage.local)
Notifications Chrome Notifications API + Alarms API
UI Theme Dark theme (GitHub-style color palette)

🏗️ Architecture

VocabBuilder follows a clean extension architecture with three main layers: background service worker, side panel UI, and data layer.

Component Architecture

Data Flow

  • User opens side panel: Extension icon click → sidepanel.js loads and queries chrome.storage.local
  • User marks "Got It": sidepanel.js → service-worker.js updates word status → Storage updated
  • User marks "Not Yet": Service worker recalculates next review date (1→3→7→14→30 days)
  • Daily reminder: Service worker (via Alarms API) → Notifications API → User notification at 9 AM
  • Badge unlocked: Service worker detects milestone → sidepanel.js displays toast notification

Manifest V3 Permissions

Permission Purpose
storage Read/write to chrome.storage.local for data persistence
alarms Schedule daily reminder notifications at 9 AM
notifications Display toast notifications (badge unlocked, daily reminder)
sidePanel Display extension UI in browser side panel

⚙️ Development Setup

Prerequisites

  • Microsoft Edge (v115+) or Google Chrome (v115+)
  • Git
  • A code editor (VS Code recommended)
  • Basic knowledge of JavaScript, HTML, and Chrome extension APIs

Installation & Running Locally

Step 1: Clone the Repository

git clone https://github.com/zozimustechnologies/vocabbuilder.git
cd vocabbuilder

Step 2: Load in Chrome/Edge

For Google Chrome:

  1. Open chrome://extensions
  2. Enable Developer mode (top right toggle)
  3. Click Load unpacked
  4. Select the vocabbuilder folder
  5. Extension appears in your toolbar → Click to open side panel

For Microsoft Edge:

  1. Open edge://extensions
  2. Enable Developer mode (bottom left toggle)
  3. Click Load unpacked
  4. Select the vocabbuilder folder
  5. Extension appears in your toolbar → Click to open side panel

Step 3: Testing & Debugging

Open the developer tools:

  • Side panel UI: Right-click on side panel → Inspect or F12
  • Service worker: Open extension details → Click Service worker link → Opens DevTools
  • Chrome Storage API: DevTools → Application → Storage → Local Storage

Step 4: Making Changes

  • Side panel changes: Edit sidepanel/sidepanel.html/css/js → Close & reopen side panel to see changes
  • Service worker changes: Edit background/service-worker.js → Click reload icon in extensions page
  • Word data changes: Edit data/words.js → Run node tests/validate-words.js to validate

Build & Icons

To regenerate extension icons:

node generate-icons.js

This creates icon assets for all required sizes (16×16, 32×32, 48×48, 128×128).

📁 Code Structure Guide

Side Panel UI

sidepanel/sidepanel.html — Main UI template with views:

  • Word of the Day: Random word card from current level
  • Got It List: Words user has learned
  • Recycle List: Words due for review
  • Badges Grid: Streak, word count, monthly challenge badges
  • Stats & Analytics: Progress overview
  • Settings: Difficulty level, daily goal, data reset

sidepanel/sidepanel.js — Event handlers and state management:

  • Load/save words from storage
  • Handle "Got It" / "Not Yet" clicks
  • Update UI based on word status
  • Manage tabs and view switching
  • Display toast notifications

Background Service Worker

background/service-worker.js — Business logic and scheduling:

  • Word Status Updates: Transition words through states (new → recycle → got it)
  • Spaced Repetition: Calculate next review date (1→3→7→14→30 days)
  • Badge System: Track milestones (streaks, word counts, monthly goals)
  • Streak Tracking: Persist daily streaks, detect breaks, reset logic
  • Daily Reminders: Use Alarms API to schedule 9 AM notifications
  • Notifications: Show toast when badges earned

✨ Key Features & Implementation

Spaced Repetition Algorithm

Core learning science: progressively increasing intervals for review to optimize memory retention.

Flow: New → Review Day 1 → Not Yet? → Review Day 3 → Not Yet? → Review Day 7 → ... → Review Day 30 → Got It! (Move to remembered list)

Implementation:

  • When user marks "Not Yet": calculate next review date based on current interval
  • Service worker compares current date with review date
  • Only words with past review dates appear in "Recycle" list
  • When user marks "Got It": word moves to "Got It" list (no more reviews)

4 Difficulty Levels

Level Target Audience Word Type
Simple Beginners, ESL learners Everyday vocabulary (200-500 word level)
Medium Intermediate learners Academic & professional vocabulary
Complex Advanced learners Literary, nuanced, sophisticated words
Competitive Exam aspirants (GMAT, GRE, LSAT, CAT) High-frequency exam words with tricky usage

Implementation: Users select level in Settings → Only words from that level appear in "Word of the Day" and quiz cycles

Badge System

Streak Badges: Unlock for consecutive daily visits

  • 🥉 3-day streak
  • 🥈 7-day streak
  • 🥇 15-day streak
  • 💎 30-day streak
  • 👑 90-day streak
  • 🌟 180-day streak
  • 🚀 365-day streak

Word Count Badges: Unlock for words learned milestones

  • 100, 250, 500, 1000, 2500, 5000, 10000 words

Monthly Challenge Badges: Complete daily goal

  • 1 week (7 days), 2 weeks (14 days), 3 weeks (21 days), full month (28 days)

Implementation: Service worker checks thresholds on each update → Toast notification → Badge added to grid

Streak Tracking & Reset Logic

  • Streak starts: First word reviewed
  • Streak continues: At least one action per day (review word, earn badge)
  • Streak breaks: More than 24 hours without activity (checked daily at 9 AM)
  • Reset logic: Service worker runs daily alarm → checks last activity date → resets if >24h gap

Daily Reminders

  • Trigger: Service worker Alarms API schedules recurring alarm at 9 AM
  • Notification: "Time to learn a new word! Open VocabBuilder."
  • Configurable: Users can disable/enable in Settings

🔌 API & Data Model Reference

Word Object Schema

{
  id: Number,                           // Unique identifier
  word: String,                         // The vocabulary word
  pronunciation: String,                // IPA format, e.g., "/prəˈpɪʃəs/"
  partOfSpeech: String,                 // "noun", "verb", "adjective", etc.
  level: "simple" | "medium" | "complex" | "competitive",
  definition: String,                   // 1-2 sentence definition
  examples: Array<String>,             // Array of usage sentences
  synonyms: Array<String>,             // Related words with similar meaning
  antonyms: Array<String>,             // Opposite meaning words
  etymology: String,                    // Word origin/history
  status: "new" | "recycle" | "got-it", // Current user's progress
  createdAt: Number,                    // Timestamp when added to user's list
  reviewDates: Array<Number>,          // History of review dates
  lastReviewDate: Number                // Last time user reviewed
}
